# Техническое задание на проект  
## «Analyzer of Technology Stacks in IT Projects»

---

## 1. Цель проекта

Цель проекта — разработка веб-сервиса для анализа и визуализации технологических стеков IT-проектов.  
Система позволяет собирать данные о используемых технологиях в проектах, анализировать их популярность, а также наглядно представлять результаты в виде таблиц и интерактивных графиков.

---

## 2. Роли пользователей

В проекте предусмотрены следующие роли пользователей:

**Гость**  
- Просматривает список проектов и используемых в них технологий  
- Анализирует статистику популярности технологий  
- Просматривает интерактивные графики и таблицы  
- Может добавлять проекты и технологии через формe  

**Администратор**  
- Управляет всеми данными через административную панель Django  
- Может добавлять, редактировать и удалять проекты, технологии и категории  

---

## 3. Модели данных (сущности)

В проекте используются следующие кастомные модели:

### Project (Проект)
Хранит информацию об IT-проектах.  
**Основные поля:**
- name — название проекта  
- description — описание проекта  
- created_at — дата создания  

---

### Technology (Технология)
Описывает отдельную технологию, используемую в проектах.  
**Основные поля:**
- name — название технологии  
- category — категория технологии (Backend, Frontend, Database и т.д.)  

---

### ProjectTechnology (Связь проекта и технологии)
Связывает проекты и технологии (Many-to-Many через промежуточную модель).  
Используется для аналитики и подсчёта популярности.  
**Основные поля:**
- project — связь с моделью Project (ForeignKey)  
- technology — связь с моделью Technology (ForeignKey)  

---

## 4. Ключевой функционал (User Stories)

- Пользователь добавляет новый проект и выбирает набор технологий через форму  
- Пользователь просматривает таблицу популярности технологий с подсчётом количества проектов  
- Пользователь фильтрует данные по выбранному проекту  
- Система автоматически агрегирует данные и подсчитывает статистику  
- Пользователь просматривает интерактивные графики популярности технологий и категорий  

---

## 5. Внешние интеграции / Аналитика

- Используется внешний API GitHub для получения статистики популярности технологий  
- Реализован собственный API проекта, возвращающий аналитические данные в формате JSON  
- Для визуализации данных используются интерактивные графики (Chart.js)

## 7. Подробные Use Case сценарии (расширенное ТЗ)

### Сценарий 1: Добавление проекта
1. Пользователь открывает главную страницу  
2. Заполняет форму добавления проекта  
3. Выбирает технологии по категориям  
4. Отправляет форму  
5. Данные сохраняются в базе  

### Сценарий 2: Анализ популярности технологий
1. Пользователь открывает страницу аналитики  
2. Система выполняет агрегацию данных через Django ORM  
3. Формируется таблица популярности технологий  
4. Отображаются интерактивные графики  

### Сценарий 3: Получение данных через API
1. Фронтенд отправляет запрос к API  
2. Сервер возвращает JSON с агрегированными данными  
3. Данные используются для динамической визуализации  

---

## 8. Алгоритмы работы ключевых функций (расширенное ТЗ)

- Агрегация данных выполняется с использованием Django ORM (Count)  
- Фильтрация проектов происходит на уровне запросов к базе данных  
- Данные для графиков формируются на сервере и передаются в формате JSON  
- Визуализация обновляется без перезагрузки страницы
